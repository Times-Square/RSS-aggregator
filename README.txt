# RSS Агрегатор Новин - Інструкція

## Опис системи
Система являє собою RSS агрегатор новин, який складається з трьох основних компонентів:
1. API сервер (FastAPI)
2. RSS парсер
3. База даних MongoDB

## Запуск системи
Система розгортається за допомогою Docker Compose. Для запуску виконайте:

```bash
docker-compose up -d
```

## Компоненти системи

### 1. API Сервер (порт 8000)
- Забезпечує REST API для отримання новин
- Endpoint для новин: http://localhost:8000/api/news
- Endpoint перевірки здоров'я системи: http://localhost:8000/api/health

### 2. RSS Парсер
- Автоматично оновлює новини кожні 30 секунд
- Зберігає новини в MongoDB
- Має вбудовану обробку помилок та повторні спроби
- Логує свою роботу для відстеження помилок

### 3. MongoDB (порт 27017)
- Зберігає всі завантажені новини
- Доступна локально через стандартний порт

## Веб-інтерфейс
- Доступний за адресою: http://localhost:8000
- Автоматично оновлюється кожні 30 секунд
- Показує статус підключення до сервера
- Має кнопку ручного оновлення

## Перевірка працездатності
1. Відкрийте http://localhost:8000 у браузері
2. Перевірте індикатор статусу (зелений - онлайн, червоний - офлайн)
3. Перевірте наявність новин на сторінці
4. Спробуйте натиснути кнопку "Оновити новини"

## Типові проблеми та їх вирішення

### Система не запускається
1. Перевірте, чи запущений Docker
2. Перевірте логи контейнерів:
```bash
docker-compose logs
```

### Новини не оновлюються
1. Перевірте логи RSS парсера:
```bash
docker-compose logs rss-parser
```
2. Перевірте доступність RSS джерел
3. Перевірте підключення до інтернету

### Веб-інтерфейс недоступний
1. Перевірте, чи запущені всі контейнери:
```bash
docker-compose ps
```
2. Перевірте логи API сервера:
```bash
docker-compose logs api
```

## Корисні команди Docker

### Перезапуск всіх сервісів
```bash
docker-compose restart
```

### Зупинка системи
```bash
docker-compose down
```

### Перегляд логів конкретного сервісу
```bash
docker-compose logs [service-name]
```

## Технічні деталі
- Система використовує Python 3.9+
- Всі залежності вказані в requirements.txt кожного сервісу
- Конфігурація системи здійснюється через змінні середовища в docker-compose.yml

## Примітки
- Система автоматично очищує старі новини
- При проблемах з кодуванням перевірте налаштування в rss_parser.py
- Система має вбудовану обробку помилок та автоматичне відновлення
